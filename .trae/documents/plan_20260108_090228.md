# 问题分析

用户要求修改生成保存到相册的字卡中相关词组的字体大小，使其略小一些。

当前代码中绘制相关词组的字体大小设置：
1. **保存字卡功能** (`saveCard`函数)：使用 `36px sans-serif`
2. **生成分享图片功能** (`generateShareImage`函数)：使用 `36px sans-serif`

# 解决方案

## 1. 修改保存字卡功能中的相关词组字体大小
- **文件**：`subgames/daily-character/index.js`
- **修改点**：在 `saveCard` 函数中，将绘制相关词组的字体大小从 `36px` 调整为 `32px`
- **同时调整**：相应调整行高和起始位置，确保布局美观

## 2. 修改生成分享图片功能中的相关词组字体大小
- **文件**：`subgames/daily-character/index.js`
- **修改点**：在 `generateShareImage` 函数中，将绘制相关词组的字体大小从 `36px` 调整为 `32px`
- **同时调整**：相应调整行高和起始位置，确保布局美观

# 预期效果

- 保存到相册的字卡中相关词组的字体大小略小一些，更加美观
- 分享图片中相关词组的字体大小也相应调整，保持一致性
- 整体布局仍然美观，不会因为字体大小的调整而显得拥挤或不协调
