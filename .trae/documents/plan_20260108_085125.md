# 问题分析

根据用户反馈，调试台显示第一个汉字有相关词组，但列表页面中相关词组没有显示出来。这说明数据加载是正常的，问题出在显示逻辑上。

当前可能导致显示问题的因素：

1. **复杂的条件判断**：`{{item.relatedPhrases && item.relatedPhrases.length > 0 ? item.relatedPhrases.join('、') : '暂无'}}`
2. **显示长度限制**：CSS中设置了`overflow: hidden; text-overflow: ellipsis; display: -webkit-box; -webkit-line-clamp: 2;`
3. **样式设置**：可能存在其他样式问题导致显示异常

# 解决方案

## 1. 简化相关词组的显示逻辑
- **文件**：`subgames/daily-character/pages/list/list.wxml`
- **修改点**：简化相关词组的显示逻辑，直接显示相关词组，移除复杂的条件判断

## 2. 移除显示长度限制
- **文件**：`subgames/daily-character/pages/list/list.wxss`
- **修改点**：移除`phrases-text`的显示长度限制，允许相关词组完整显示

## 3. 调整样式设置
- **文件**：`subgames/daily-character/pages/list/list.wxss`
- **修改点**：确保相关词组的容器有足够的空间，调整行高和间距

# 预期效果

- 相关词组能够正常显示，不再受长度限制
- 简化的显示逻辑减少了出错的可能性
- 充分利用可用的显示空间，显示完整的相关词组信息
