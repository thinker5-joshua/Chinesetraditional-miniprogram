# 守护神兽功能改进方案

## 设计目标
模仿二十八星宿探秘的设计模式，优化守护神兽功能，实现：
1. 首页直接展示脊兽列表信息
2. 点击脊兽查看详细信息
3. 顶部展示已抽取的专属守护神兽
4. 支持重复查看已抽取神兽的海报
5. 重新抽取时替换已存储的神兽信息

## 实现方案

### 1. 首页布局改进
- **顶部区域**：展示已抽取的守护神兽信息（包含图片、名称、查看海报按钮）
- **中部按钮**：添加"抽取您的专属守护神兽"按钮
- **下部区域**：展示完整的脊兽列表，采用卡片式布局

### 2. 本地存储实现
- 使用`wx.setStorageSync`/`wx.getStorageSync`存储已抽取神兽信息
- 存储结构：`{ beastId, name, image, posterPath, extractedTime }`
- 页面加载时读取本地存储，更新UI显示

### 3. 功能交互优化
- **抽取流程**：抽取完成后返回首页，自动更新已抽取神兽信息
- **海报查看**：已抽取神兽区域添加"查看海报"按钮，直接跳转至海报页
- **重新抽取**：再次抽取时，替换本地存储的神兽信息
- **脊兽详情**：点击列表中的脊兽，跳转至详情页查看完整信息

### 4. 页面结构调整
- 修改`index.wxml`：重新组织布局，添加已抽取神兽展示区和脊兽列表
- 更新`index.js`：添加本地存储读写逻辑和页面数据管理
- 保持`knowledge/list.js`和`knowledge/detail.js`的现有功能
- 优化`draw.js`：抽取完成后返回首页并更新本地存储

### 5. 设计风格统一
- 采用与二十八星宿探秘相似的卡片式布局
- 保持一致的颜色主题和交互效果
- 优化响应式设计，适配不同屏幕尺寸

## 预期效果
- 用户进入首页可直接查看所有脊兽信息
- 已抽取的守护神兽突出展示在顶部
- 支持便捷地重新抽取或查看已抽取神兽的海报
- 整体设计与二十八星宿探秘保持一致，提升用户体验

## 技术要点
- 微信小程序本地存储API的使用
- 页面间数据传递和状态管理
- 响应式布局设计
- 云存储图片的处理和展示
- 统一的视觉设计语言
