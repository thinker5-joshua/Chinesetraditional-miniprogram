# 相关词组显示问题修复计划

## 问题分析

根据用户反馈，数据是正确的，但相关词组在列表页和测试页中不显示。通过分析代码：

1. **列表页**：
   - WXML：使用了正确的条件渲染语法 `{{item.relatedPhrases ? item.relatedPhrases.join('、') : '暂无'}}`
   - WXSS：`.phrases-text` 颜色为 `#666`，字体大小为 `32rpx`

2. **测试页**：
   - WXML：使用了正确的条件渲染语法 `{{testQuestions[currentQuestionIndex].relatedPhrases ? testQuestions[currentQuestionIndex].relatedPhrases.join('、') : '暂无'}}`
   - WXSS：`.phrases-text` 颜色为 `#888`，字体大小为 `28rpx`

## 修复方案

### 1. 调整相关词组颜色为红色

修改两个页面的 `.phrases-text` 样式，将颜色改为红色，以便于观察显示情况：

**列表页** (`subgames/daily-character/pages/list/list.wxss`)：
```wxss
.phrases-text {
  flex: 1;
  font-size: 32rpx;
  color: #ff0000; /* 改为红色 */
  line-height: 1.4;
}
```

**测试页** (`subgames/daily-character/pages/test/index.wxss`)：
```wxss
.phrases-text {
  font-size: 28rpx;
  color: #ff0000; /* 改为红色 */
}
```

### 2. 增强显示效果

为确保相关词组能够清晰显示，增加以下样式调整：

**列表页**：
- 增加 `.phrases-text` 的 `font-weight` 为 `500`，使文字更醒目
- 确保 `.item-body` 容器有足够的高度显示内容

**测试页**：
- 增加 `.phrases-text` 的 `font-weight` 为 `500`
- 确保 `.related-phrases` 容器有足够的 padding

### 3. 检查可能的遮挡问题

- 检查 `scroll-view` 的高度设置是否合理
- 检查容器的 `overflow` 属性是否影响显示
- 确保没有其他元素的 `z-index` 高于相关词组
