## 问题分析
当前需要解决的问题：
1. 阻止页面上下滚动，但保留小程序右滑退出功能
2. 确保字卡可以正常左右滑动切换
3. 避免影响小程序的系统级交互

## 解决方案

1. **优化页面布局**：
   - 确保页面内容不会超出视口高度
   - 使用固定定位或flex布局，避免页面出现滚动条
   - 给主容器添加 `overflow: hidden`，防止内容溢出滚动

2. **精确控制触摸事件**：
   - 只在字卡区域处理滑动事件
   - 保留页面边缘的右滑退出功能
   - 区分字卡滑动和页面边缘滑动

3. **优化触摸事件处理**：
   - 在字卡区域，水平滑动只触发字卡切换，不触发页面滚动
   - 垂直滑动不触发任何滚动
   - 页面边缘区域保留右滑退出功能

## 具体实现

1. 修改 `index.wxss`：
   ```css
   /* 确保页面内容不会超出视口高度 */
   .app-container {
     overflow: hidden;
     height: 100vh;
     display: flex;
     flex-direction: column;
   }
   
   .main-content {
     overflow: hidden;
     flex: 1;
   }
   ```

2. 修改 `index.js`，优化触摸事件处理：
   - 检测触摸起始位置，如果在页面右边缘（比如右侧20px以内），不阻止默认行为
   - 否则，完全阻止垂直滚动，只允许水平滑动字卡
   - 保留现有的字卡滑动切换逻辑

3. 确保字卡的触摸事件正常工作：
   - 保持现有的字卡触摸事件处理逻辑
   - 确保字卡的滑动切换功能正常

## 预期效果
- 页面不会上下滚动
- 字卡可以正常左右滑动切换
- 小程序的右滑退出功能正常工作
- 提高用户体验，让字卡滑动更加流畅
