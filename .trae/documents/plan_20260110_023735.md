# 问题分析

**用户反馈**：
- 提取字的逻辑有误：不应该提取词语中的第一个汉字，而是提取易错字，一般是正确读音前的那一个字
- 需要增加原始文本属性，方便后期检查

**文件内容分析**：
- 文件包含大量易读错字例子，格式多样
- 94-194行：100个易读错字，格式为"序号、词语 正确读音 错误读音：（错误读音）"
- 其他部分：更详细的易读错字列表，包含词语、正确读音和解释

# 解决方案

**修正后的数据提取方案**：

1. **创建Python提取脚本**：
   - 编写脚本自动解析"易读错字.txt"文件
   - 提取规则：
     - 提取易错字（词语中的关键字，通常是正确读音对应的字）
     - 提取括号中的正确读音
     - 提取文件中标记的错误读音
     - 保留原词语作为相关词组
     - 保存原始文本，方便后期检查
   - 支持多种格式的解析

2. **提取步骤**：
   - 优先处理94-194行的100个易读错字，这些包含错误读音信息
   - 然后处理其他格式的数据
   - 去重处理，避免重复条目
   - 对于没有错误读音的条目，留空错误读音字段

3. **数据存储**：
   - 创建JSON文件存储提取的数据
   - 格式：
     ```json
     [
       {
         "char": "易错字",
         "correctPronunciation": "正确读音",
         "wrongPronunciation": "错误读音",
         "relatedPhrases": ["相关词组"],
         "originalText": "原始文本"
       }
     ]
     ```

4. **验证和修改**：
   - 生成初始数据列表后，让用户检查和修改
   - 根据用户反馈调整提取规则
   - 逐步完善数据，添加其他缺失的属性

**技术优势**：
- 自动化提取，减少人工工作量
- 支持多种格式的解析，覆盖文件中的各种数据形式
- 去重处理，确保数据的唯一性
- 结构化存储，便于后续的修改和扩展
- 包含错误读音和原始文本信息，符合每日一字的功能需求

**预期成果**：
- 生成一个包含易错字、正确读音、错误读音、相关词组和原始文本的JSON数据文件
- 数据格式规范，便于导入到每日一字的数据文件中
- 为后续完善其他属性（如详细解释等）打下基础
