## 每日一字次要按钮区修改方案

### 一、需求分析
1. 将现有次要按钮区改为仅显示图标的按钮
2. 鼠标悬停时显示文字说明
3. 在现有3个按钮基础上增加2个新按钮，共5个
4. 为每个功能选择合适的图标
5. 保持美观简洁的排版

### 二、按钮功能与图标设计
| 功能 | 图标 | 说明 | 状态 |
|------|------|------|------|
| 保存 | 📥 | 生成字卡图像并保存到照片 | 已实现 |
| 列表 | 📋 | 显示易读错字整体列表，可切换收藏列表 | 新增 |
| 收藏 | 🤍/❤️ | 收藏或取消收藏（根据状态切换） | 已实现 |
| 测试 | 📝 | 测试易读错字 | 已实现 |
| 设置 | ⚙️ | 设置字卡显示难度和测试难度 | 新增 |

### 三、布局调整方案
1. **容器布局**：保持现有flex布局，修改为`justify-content: center`，调整间距为25rpx
2. **按钮样式**：
   - 改为圆形按钮，直径70rpx
   - 仅显示图标，移除按钮内文字
   - 图标大小32rpx，居中显示
3. **悬停效果**：
   - 鼠标悬停时显示文字提示
   - 提示文字位于按钮下方
   - 平滑过渡动画
4. **新增按钮**：添加列表和设置按钮，保持相同样式

### 四、样式修改要点
1. **按钮容器**：
   - 调整`.secondary-btn`为圆形
   - 移除`flex-direction: row`，改为垂直排列
   - 增加`position: relative`用于提示文字定位
2. **图标样式**：
   - 增大`.secondary-icon`至32rpx
   - 移除`margin-right`
3. **文字提示**：
   - 创建新样式`.tooltip`用于悬停文字
   - 初始隐藏，悬停时显示
   - 位于按钮正下方
4. **响应式调整**：
   - 保持在小屏幕上的良好显示
   - 调整间距和大小

### 五、功能实现
1. **新增事件处理函数**：
   - `showCharacterList()`：显示易读错字列表
   - `openSettings()`：打开设置页面
2. **未实现功能提示**：
   - 使用`wx.showToast`显示"功能开发中"提示
3. **保持现有功能不变**：
   - 保存、收藏、测试功能保持原有实现

### 六、文件修改清单
1. **index.wxml**：
   - 修改现有按钮结构
   - 添加两个新按钮
   - 添加文字提示结构
2. **index.wxss**：
   - 修改按钮样式
   - 添加悬停效果
   - 添加文字提示样式
3. **index.js**：
   - 新增两个事件处理函数
   - 添加功能开发中提示

### 七、预期效果
1. 按钮区更加简洁美观
2. 图标直观易懂
3. 悬停显示文字增强可用性
4. 新增功能按钮为后续扩展预留
5. 整体风格与现有设计保持一致

### 八、注意事项
1. 确保在不同设备上的兼容性
2. 保持触摸友好的按钮大小
3. 确保文字提示清晰可见
4. 保持平滑的动画效果
5. 未实现功能要有明确提示
