## 问题分析
当前字卡滑动时页面其他部分也会滚动的原因是：
- 用户无法保证完全水平滑动，轻微的上下滚动就会导致页面整体滚动
- 现有的方向检测逻辑只在水平滑动距离大于垂直滑动距离时才阻止默认滚动行为
- 当用户滑动角度接近45度时，可能会触发页面滚动

## 解决方案

1. **使用CSS触摸属性**：在字卡容器上添加`touch-action: pan-y;`，允许垂直滚动但阻止水平滚动的默认行为
2. **实现触摸锁定机制**：
   - 当检测到用户开始水平滑动时，锁定触摸事件
   - 在后续的滑动中一直阻止默认滚动行为，直到触摸结束
   - 这样即使有轻微的上下滚动，也不会触发页面整体滚动
3. **优化事件绑定**：确保触摸事件只在字卡区域生效

## 具体实现

1. 修改`index.wxss`，在字卡容器上添加触摸属性：
   ```css
   .card {
     touch-action: pan-y;
   }
   ```

2. 修改`index.js`，实现触摸锁定机制：
   - 添加`isTouchLocked`状态变量
   - 在`onTouchStart`中初始化
   - 在`onTouchMove`中检测并锁定触摸方向
   - 在`onTouchEnd`中重置锁定状态

## 预期效果
- 左右滑动字卡时，只有字卡切换，页面其他部分保持不动
- 即使有轻微的上下滚动，也不会触发页面整体滚动
- 滑动操作更加流畅，提高用户体验
