## 全面修复data-optimized.js文件中的多行字符串语法错误

### 问题分析
- data-optimized.js文件中存在多处多行字符串语法错误
- 这些错误主要出现在explanation字段中，包含了实际的换行符
- JavaScript中字符串不能直接跨行，需要使用适当的语法处理

### 修复方案

1. **创建自动修复脚本**：
   - 创建Python脚本`fix_multiline_strings.py`
   - 读取data-optimized.js文件
   - 识别所有包含多行字符串的字段
   - 将多行字符串转换为使用`\n`转义的单行字符串
   - 保存修复后的文件

2. **具体修复逻辑**：
   - 遍历文件中的每个字符串字面量
   - 检测字符串中是否包含实际的换行符
   - 将实际换行符替换为`\n`转义序列
   - 确保字符串保持在单行内

3. **验证修复**：
   - 运行修复脚本
   - 检查修复后的文件是否无语法错误
   - 验证数据结构保持完整
   - 确保所有字段内容正确保留

4. **备份策略**：
   - 在修复前自动创建文件备份
   - 确保数据安全，防止意外丢失

### 技术细节
- **修复前**：
  ```javascript
  "explanation": "1. 穿在轮子中间的圆柱形物件：车轴。轴承。
  2. 像车轴的：轴心。轴线。
  3. 把平面或立体分成对称部分的直线：轴线。"
  ```

- **修复后**：
  ```javascript
  "explanation": "1. 穿在轮子中间的圆柱形物件：车轴。轴承。\n2. 像车轴的：轴心。轴线。\n3. 把平面或立体分成对称部分的直线：轴线。"
  ```

### 预期结果
- 修复后data-optimized.js文件无语法错误
- 所有多行字符串都被正确转换为单行字符串
- 数据结构保持完整，所有字段内容正确保留
- 文件可以正常被JavaScript解析和使用
