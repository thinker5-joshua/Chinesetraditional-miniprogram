# 列表页相关词组标题不折行修复计划

## 问题分析

**现象**：当列表页中相关词组较多需要折行显示时，前面的"相关词组："标题也会折成两行，影响美观。

**根本原因**：
- `.item-body` 使用了 `display: flex` 布局
- `.phrases-label` 是一个文本元素，默认会根据空间调整换行
- 当词组内容较多时，flex 布局会压缩 label 的空间，导致其换行

## 修复方案

修改 `subgames/daily-character/pages/list/list.wxss` 文件，为 `.phrases-label` 添加不换行的样式：

```wxss
.phrases-label {
  font-size: 32rpx;
  color: #8B4513;
  font-weight: 500;
  margin-right: 10rpx;
  white-space: nowrap; /* 防止换行 */
  flex-shrink: 0; /* 防止 flex 布局压缩 */
}
```

## 修复原理

1. **white-space: nowrap**：强制文本在同一行显示，不进行换行
2. **flex-shrink: 0**：在 flex 布局中，防止元素被压缩，保持其原始宽度

## 实现步骤

1. 打开 `subgames/daily-character/pages/list/list.wxss` 文件
2. 找到 `.phrases-label` 的样式定义
3. 添加 `white-space: nowrap;` 和 `flex-shrink: 0;` 属性
4. 保存文件并在微信开发者工具中预览效果
5. 验证标题是否不再换行
6. 确认修复成功后完成任务
