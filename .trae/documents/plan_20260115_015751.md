## 修复错误原因分类显示机制

### 问题分析
- **代码实现**：在`index.js`的`getErrorReasonText`方法中，错误原因分类映射与新规则不匹配
- **数据文件**：数据文件中使用的是1-8的分类规则，与新规则一致
- **具体差异**：
  - 代码只处理了1-6的情况，缺少7-8的处理
  - 分类名称和编号对应关系不一致

### 修复方案

1. **更新index.js中的错误原因映射**：
   - 修改`getErrorReasonText`方法中的`errorReasonMap`对象
   - 按照新规则更新分类名称和编号对应关系
   - 添加缺失的7和8分类

2. **具体修改内容**：
   - 将当前的映射：
     ```javascript
     const errorReasonMap = {
       1: '形声字误读',
       2: '形近字混淆',  
       3: '音近字/同音字混淆',
       4: '多音字误读',

     ```
   
   - 更新为新规则：
     ```javascript
     const errorReasonMap = {
     ```
